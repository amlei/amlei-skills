# job-market-research

## 输入

1. **求职岗位**：目标岗位方向。
2. **目标城市**：意向工作城市。

## 交付

就业市场报告，写入 `.md` 文件。**若已有同名文件存在，先备份（带时间戳）再写入，不直接覆盖。** 返回：**文件路径** + **简短摘要**（薪资范围、核心能力概览）。

报告内容：
- **薪资范围**：该城市该岗位的薪资区间（校招/社招区分）
- **能力总览**：该岗位需要具备的硬技能（技术栈/工具/框架）和软技能（沟通/协作/管理等）
- **城市分布**：该岗位的主要就业城市和机会密度
- **行业趋势与最新动向**：该岗位 / 行业的发展方向、热点领域，以及**最新演进**——哪些能力正从"加分"变成"刚需"、岗位职责是否在扩展（如向综合能力靠拢：开发要懂业务 / 产品 / 设计）、近期出现的新要求。这部分是下游简历定位和评估的关键依据，要具体、有引用依据，不要泛泛罗列热点词。

## 规则

1. **联网搜索客观数据**：从招聘平台薪资统计、行业报告、公开招聘信息中提取，不凭记忆编造。
2. **硬技能 + 软技能全覆盖**：不局限于技术栈，也覆盖岗位需要的软性能力。
3. **标注来源**：每条关键数据标注信息来源。
4. **看行业普遍期望**：不局限于单一公司，呈现行业对该岗位的整体期望。
5. **区分层级**：初级/中级/高级岗位的能力要求和薪资差异分开呈现。
6. **抓最新演进，不只静态快照**：行业趋势要呈现岗位的"演进方向"——什么能力在变成刚需、职责是否在扩展、近期新要求，而非只堆当前热点词。这是下游简历能否跟得上岗位最新动向的关键。

## 验收自检

交付前逐项确认，任何一项不通过则补齐后再交付：

- [ ] 薪资范围有数据支撑，非凭空估计
- [ ] 硬技能和软技能均已覆盖
- [ ] 信息来源已标注
- [ ] 行业趋势有引用依据
- [ ] `.md` 文件已写入磁盘
- [ ] 返回内容只有文件路径 + 摘要
