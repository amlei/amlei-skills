# 写简历（0→1）

从零跟用户一起写目标岗位简历。

1. **聊清目标**：求职岗位、意向城市、工作年限。
2. **岗位调研**（优先）：基于用户给到的公司和 JD（见 SKILL.md JD 调研原则），联网搜索该岗位的招聘要求、候选人画像、潜在/隐性能力需求——以具体公司的 JD 为锚，先摸清"对方要什么人"，再回头跟用户讨论。
3. **查个人资料**：`memory.py find --tag <岗位方向>` 拉相关能力 / 项目，按结果取舍。
4. **挖素材**：围绕岗位关键词，**主动问用户**——聊项目里最有成就感的事、遇到的最大困难怎么解决、同事怎么看你的优势、你是怎么学某个技能的。从用户的回答中提取 STAR 素材和量化线索，用户说不清的帮忙追问具体数字。
5. **共创**（默认逐节）：按 个人简介 → 经历 → 技能 顺序，每节基于挖到的素材 + JD出一版给用户确认 / 改，再下一节。要快就切「先整篇草稿再改」。
6. **落盘**：写到 `resume/{姓名}/{求职岗位}/简历.md`，给用户。
7. 经历用 STAR + 量化，只放与岗位相关的；个人资料里的项目地址优先访问核实数据。
