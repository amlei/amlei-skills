# amlei-resume

和用户一起写、改、迭代简历内容，再装配 HTML 导出 PDF。**个人资料**是素材库（多查、写入谨慎）；**简历**是工作产物（项目级持久化、可反复改）。

核心原则：**不直接问"你要改哪"——通过聊项目、聊做事方式、聊自我认知，帮用户挖出他们自己都忘了的亮点，再落成简历文字。**

## 开始前：检查现有简历

每次写/改简历前，先检查 `resume/{姓名}/{求职岗位}/简历.md` 是否存在（项目级文件）：

- **有简历** → 问用户基于当前版本需要做哪些调整。
- **无简历** → 问用户是否有旧版简历（Word / PDF）需要导入；有则走[简历导入](#简历导入)流程；没有就从零开始。

确认简历状态后，查个人资料里的求职偏好（`preference` 无参显示）——城市、薪资、经验等作为岗位调研的锚点。再问用户要他觉得不错的目标公司 JD。

## 岗位调研（通用）

用户给出 JD（文字/链接/文件）后，**立即将 JD 全文写入 `resume/{姓名}/{求职岗位}/jd.md`**（一个岗位一个 JD 文件，不修改）。每次写/改前都先确认 JD 是最新的、已落盘。

拿到 JD 后，和用户围绕这几方面讨论，帮用户自我定位：

1. **聊 JD 描述**：把 JD 里列的要求一条条跟用户过——"这个岗位要求熟练 X 技术，你之前用过吗？"、"它提到需要 Y 领域经验，你有哪些相关的？" 把 JD 语言翻译成用户能对号入座的话。
2. **网络搜索岗位画像**：基于 JD 搜索该岗位的人才定位——这个岗位需要什么样背景的人、典型候选人画像、常见能力模型和隐性要求。不局限于这一家公司，而是看行业对这个岗位的普遍期望。
3. **引导自我认知**：从 JD 的关键词出发，帮用户发现自己的对应能力和亮点，而非直接问"你会什么"。
4. **聊现状与规划**：了解用户当前状态（城市、工作、年限），询问未来规划——"你现在在广州，下一份工作打算去哪个城市？""你希望这份工作带来什么改变？"

## 项目讨论（通用）

写简历和改简历都需要挖掘项目经历。详见 [references/projects.md](references/projects.md)。

## 能力

- **写（0→1）简历**：从零跟用户一起写目标岗位简历。详见 [references/write.md](references/write.md)。
- **改简历**：通过聊经历、项目故事来激发补充、挖掘沉默亮点。详见 [references/revise.md](references/revise.md)。
- **资料联动**：个人资料一更新，**主动**告诉用户「要不要把它补进 X 岗位简历」——确认才动。
- **联网索迹**：根据 JD 联网搜索岗位人才定位——行业对该岗位的普遍期望和能力模型。
- **导出**：用户要求导出时，校验格式 → 选主题 → 装配 HTML → 导出 PDF。

## 简历存储

简历是普通 Markdown，持久化在项目级：

```
resume/{姓名}/{求职岗位}/简历.md
```

改写简历就读这份；换岗就开新的 `{求职岗位}/` 目录存新版（原岗版本保留）。**用户没说"可以/就这样"之前，绝不写入简历文件。**

## 简历格式

校验脚本会查的硬规则：

| 元素 | 写法 |
|------|------|
| 首模块 | `# self-intro`（必须是第一个 `#`），下用 `key: value` 放 name / role / gender / location / phone / email / avatar / links 等 |
| 模块 | `# 模块名`（出现顺序 = 简历模块顺序） |
| 简介 | 模块下纯文本段 |
| 标签 | 模块下「标签, 标签」单行（研究兴趣 / 方向） |
| 经历 | `## 机构 \| 角色/方向`（无 `\|` 则只有机构名） |
| 经历日期 | `date: 2024.07 — 2025.03` |
| 经历补充 | `meta: GPA 3.9/4.0 · 排名 1/60`（关键数字用 `**`） |
| 经历要点 | `- 成就（STAR + 量化）` |
| 技能 | `- 类别: 值` 或 `类别: A · B · C` |

`name:` 必填；`avatar:` 给照片相对路径；`links` 放可线上展示自己的链接，如个人博客、飞书文档、作品集等（key: value）。完整范例：[sample-resume.md](assets/sample-resume.md)。

## 简历导入

用户给旧简历文件（.docx / .pdf），用 markitdown 读取转 Markdown：

```bash
markitdown <文件路径> -o <输出路径.md>
```

将转换结果重排成简历格式 → 写到 `resume/{姓名}/{求职岗位}/简历.md`（自动建目录）。证件照用 `scripts/extract_avatar.py` 抽取放进同目录，MD 用 `avatar:` 引用。若要写入个人资料走「评估 + 确认」。

## 易错点

- **残留 `{{占位符}}`**：组件模板占位符必须全替换（校验兜底）。
- **`# self-intro` 必须是首个 `#`**，渲染成 Header 而非 SectionHead。
- **`data-stick="1"` 别漏**：SectionHead 必带。
- **联系方式按实际字段拼**：没有的字段连同分隔符一起删。
- **`avatar:` 有就 `<img>`**：没照片用 icons.md 占位 SVG。
- **整篇只用所选主题的组件**，不从别的主题借。
- **标题层级只用 `#`/`##`**。

## 添加新主题

新主题可直接编写或从参考简历 HTML 转换（规范见 [references/themes/theme-index.md](references/themes/theme-index.md) 末尾）。写完用 `assets/sample-resume.md` 渲染 → 校验 → 套壳 → 浏览器打开预览核对，确认后登记到 theme-index。
