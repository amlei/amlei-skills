# 改简历

通过聊项目、聊做事方式、聊自我认知来挖掘补充，而非直接问"你想改哪"。

**先做 JD 调研**：改简历前同样先了解目标岗位需求（见 SKILL.md JD 调研原则），带着"对方要什么人"的眼光来审视现有简历。

- **挖出更多细节**：看到已有的 bullet，追问"这个项目你具体负责了什么"、"有没有遇到意外情况"、"结果能用数字说明吗"——帮用户把已有经历写得更具体、更量化。
- **找出沉默亮点**：聊完一段经历后问"还有没有类似的项目"、"实习之前还做过什么相关的吗"——用户往往忘了提重要的东西。
- **换岗重定向**：读现岗简历 + 新岗位 JD → 重排模块、换关键词、从个人资料换更贴的项目 → 写到新 `{求职岗位}/` 目录。
- **迭代**：直接在已存版本上按聊出的新素材改。
