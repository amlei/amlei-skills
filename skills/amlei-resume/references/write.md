# 写简历（0→1）

从零跟用户一起写目标岗位简历。

1. **聊清目标**：求职岗位、意向城市、工作年限。
2. **保存 JD + 岗位调研**（优先）：用户给出 JD（文字/链接/文件）后，**立即将 JD 全文写入 `resume/{姓名}/{求职岗位}/jd.md`**，然后联网搜索该岗位的人才定位——行业普遍期望的能力模型和候选人画像，不局限于这一家公司。
3. **查个人资料**：`profile.py find --tag <岗位方向>` 拉相关能力 / 项目，按结果取舍。
4. **挖素材**：围绕岗位关键词，**主动问用户**——按 [projects.md](projects.md) 的流程聊项目经历（含素材持久化：区分公司/个人项目、让用户给文件、没有就聊、STAR 素材写入 `materials.md`）。从用户的回答中提取 STAR 素材和量化线索，用户说不清的帮忙追问具体数字。
5. **共创**（默认逐节）：按 个人简介 → 经历 → 技能 顺序，每节基于挖到的素材 + JD出一版给用户确认 / 改，再下一节。要快就切「先整篇草稿再改」。
6. **用户确认后再保存**：每版内容经用户明确认可后，再写到 `resume/{姓名}/{求职岗位}/简历.md`。用户没说"可以/就这样"之前，绝不写入。
7. 经历用 STAR + 量化，只放与岗位相关的；个人资料里的项目地址优先访问核实数据。
