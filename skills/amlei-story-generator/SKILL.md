# 故事生成

把任意一个主题，写成一篇**有钩子、有故事、有共鸣**的长文，帮读者深度吸收。不是说明文，是 **始于困惑、陷于故事、归于共鸣** 的叙事。

**标准**：开头让人想往下读；正文让人记住金句和落点；结尾让人把故事里的东西接回自己的人生。

## 取材
每次都先联网搜索——无一例外。人名、年代、金句、数据这类硬事实，记忆不可靠，必须搜到再写。

按**素材类型**查齐所需（类型判定与骨架见 [source-frameworks.md](resources/source-frameworks.md)）：

- **书**：作者(为何有资格)、核心论点、核心内容、核心观点、金句
- **人**：身份为何重要、决定性瞬间、性格成因、关键抉择、代价、金句
- **事**：发生了什么、为何发生、关键人物、转折点、影响、金句
- **物**：它是什么、怎么诞生、演进历程、为何重要、改变了什么、金句
- **理**：它是什么、解决什么问题、关键例证、争议/边界、金句
- **案**：案例本身、反常/启发性、背后机制、可迁移洞察、金句

技巧：先 `主题 + 背景 / 书评 / 访谈` 摸全局，再 `金句关键词 + 主题` 逐句核原句；中文素材搜中文，外文原句交叉搜原文。

> **铁律：金句必须真实。** 搜原文核对，否则不引；宁可少引，绝不编造。翻译与呈现细则见 [narrative-craft.md](resources/narrative-craft.md)「金句定点投放」。

## 行为方式
整篇不是一次成稿的流水线，而是一边写一边回头改——三个阶段，每个阶段都"没到位就回去重做"。

**取材** · 先搜，搜齐再说。按素材类型把要素查全；每条金句核到原文，搜不到就换关键词再搜或干脆不引。要素没齐、金句没核到，就继续搜，不进下一步。这时一并判定素材类型，取对应骨架。

**起稿** · 先立钩子，再铺主干。开篇撕开认知缺口、引出钩子标题；正文铺成 3–7 个序号小节，标题是钩子不是标签。每节场景先行、代入读者、金句定点。手艺正反例见 [narrative-craft.md](resources/narrative-craft.md)。

**打磨** · 通读整篇：三步法则（始于困惑 → 陷于故事 → 归于共鸣）成不成立？金句定没定点？结尾有没有回到开篇、接回读者生活、别用总结腔？哪节不成立，回起稿改那节。

自评每项都达标后，送评判者。

## 送评：评判者
自评达标后，**起一个评判者子 agent 独立打分**——独立上下文、不共享草稿，避免自评偏袒：

- 用你当前环境里「创建子 agent」的方式起一个**独立上下文**的评判者——**具体调用哪个工具不在本规则里固定**（不同 Agent 平台机制不同，由 LLM 自行选用）。
- 传给它：主题 + 全文 + [rubric.md](resources/rubric.md)
- 它输出：逐维得分表 → 总分 → 达标 / 不达标 → 不达标则按优先级列出具体修改点（定位到小节 / 句子）

**达标线**：总分 ≥ 90，且全部硬性门槛通过（清单见 [rubric.md](resources/rubric.md)）。

**不达标**：按修改点回去改，改完**重新送评**——总共**最多评分 3 轮**（初次送评 + 至多 2 次重评）；第 3 轮仍不达标，就交付当前最优版本 + 评判者意见。

## 输出规格
- **长度**：2000–4000 字，宁可少而准，不注水。
- **结构**：`#` 钩子标题（**整篇只有一个 `#`**）→ `## 01 / 02 / 03…` 序号小节（用 `##`，不用 `#`）→ 共鸣收尾。
- **金句**：真实、定点、不堆砌。
- **交付**：默认输出在对话里；要存盘写到 `故事-<主题>.md`。

## 速查：三步法则
| 阶段 | 读者状态 | 手法 |
|---|---|---|
| 始于困惑 | "为什么？" | 钩子开篇，撕开缺口 |
| 陷于故事 | "后来呢？我也是" | 场景 → 好奇 → 术语；金句定点 |
| 归于共鸣 | "原来如此" | 回到开篇，接回读者 |
