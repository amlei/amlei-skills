# text-polish

## 输入

1. **待润色文件**：需要润色的文本文件路径，或直接提供的文本内容。
2. **润色目标**（可选）：希望达成的效果（如"更简洁""更有冲击力""更正式""更口语化"）。

## 交付

润色后的文件，写入 `.md`（或与输入相同格式）。**若已有同名文件存在，先备份（带时间戳）再写入，不直接覆盖。** 返回：**文件路径** + **简短摘要**（主要改了哪些方面、有无结构性调整建议）。

## 规则

1. **逐行分析**：从头到尾逐行阅读原文，理解每行在上下文中的角色和意图，不跳读、不批量替换。
2. **按目标选词**：根据润色目标，为每处表达选择最合适的词语和描述方式。同一意思在不同语境下措辞不同——正式场合用书面语，技术文档用精准术语，叙事文本用生动动词。
3. **不改原意**：润色只改表达方式，不改事实、不编造内容、不增删信息点。每处改动都能对应到原文意图。
4. **上下文一致**：修改某处时同步考虑上下文——术语全文统一、语气连贯、前后呼应，不孤立地改一句导致前后割裂。
5. **消除冗余**：同义重复、跨段重复的内容合并；对表达意图无贡献的词句删减。
6. **保持结构**：不改变文档的标题层级、模块顺序、段落划分。如需调整结构，在摘要中标注建议但不在文件中执行。
7. **保留专有内容**：数字、专有名词、技术术语、引文原样保留，不做"优化"。

## 验收自检

交付前逐项确认，任何一项不通过则补齐后再交付：

- [ ] 逐行分析过全文，无遗漏
- [ ] 未改变任何事实和数据
- [ ] 未编造新内容
- [ ] 用词与上下文语境一致，术语全文统一
- [ ] 标题层级和模块结构未变
- [ ] 数字、专有名词、引文原样保留
- [ ] 文件已写入磁盘
- [ ] 返回内容只有文件路径 + 摘要
